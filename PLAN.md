# Plan: krunch UI re-imagining — "Mission Control" terminal cockpit
_Locked via grill — by Claude + kyle.dougan_

## Goal
Completely re-imagine krunch's frontend from the calm, old-world "Deliberation Chamber" into a **fun, dense, effect-heavy terminal/mission-control cockpit built for power users**. The deliberation domain (seats, mediator, rounds, stances/confidence, rulings, verdict, awaiting-user Q&A, export) and the Tauri + Vue 3 + Tailwind v4 + Pinia stack are retained; the visual language, layout, and interaction model are rebuilt from zero. The app should read like a tactical console where a panel of LLM agents argues to a verdict in real time — keyboard-native, telemetry-rich, alive with ambient CRT atmosphere and punchy event-driven effects. A small, surgical Rust change surfaces real per-seat token usage so the console shows true numbers.

## Approach
1. **Design tokens & atmosphere (`src/style.css`).** Replace the brass/charcoal OKLCH system with a multi-color tactical terminal palette: near-black base + status spectrum — green (stance/consensus), amber (streaming/active), red (deadlock/abstain), cyan (mediator/system). Rewire the shadcn aliases to the new hues. Add ambient layer primitives: faint scanline overlay, soft phosphor text-glow, blinking cursor, subtle vignette — all always-on but tuned for long-read legibility. Keep the existing film-grain hook. Define effect keyframes: `boot`, `glitch`, `type-on`, `latch`, `shake`, `flicker`. All gated behind `prefers-reduced-motion`.
2. **Typography.** Drop Young Serif + Hanken Grotesk. Make Fragment Mono the near-universal UI font. Add ONE blocky display face for big moments (wordmark, verdict, deadlock banners): **Departure Mono** (SIL OFL 1.1, self-hosted woff2, ≤ ~40 KB). Update `fonts.css` + `@theme` font vars.
3. **Cockpit shell (rewrite `App.vue`).** Replace the three full-screen swap with ONE persistent tiled console:
   - **Status bar** (always visible): session id, elapsed clock, round NN/MM (clamped to `max_rounds` — finalization never shows `NN/<max+1>`), live **convergence readout**, aggregate telemetry (token totals labeled `exact` only when every completion reported both counts, else `partial`; `~tok/s` approximate; est. $ or `—`), and a `⌘K` hint.
   - **Main region** morphs by `store.phase`: `setup` → dense config pane; `room` → live panel grid + mediator + telemetry/log rail; `verdict` → ruling overlay. No full-screen transitions — panes swap/morph in place with a glitch transition.
   - **Command palette** (`⌘K`/`Ctrl-K`): NO shadcn-vue `Command` exists in this repo, so build it from the existing `ui/dialog` + reka `ListboxRoot`/combobox (or add the shadcn-vue command block via CLI). Discoverable shortcuts (c=convene, a=add seat, e=export, ?=help, 1–6 focus seat, Esc) live in a **centralized keybinding module** with guards for: `contenteditable`, `<select>`, IME `event.isComposing`, modifier keys, phase-aware enablement (a shortcut invalid in the current phase is disabled), and focus restoration after the palette closes.
4. **Config pane (rebuild `SetupScreen` + `SeatEditor`).** Dense terminal form: mono labels, inline validation as a "preflight" checklist, keyboard-addable seats. Preserve every field (provider, base_url, model, system_prompt, sampling) and the 3 interaction modes. `loadDemoPanel` kept.
5. **Live panel grid (rebuild `RoomScreen` + `SeatCard` + `MediatorPanel`).** Each seat = a console panel streaming type-on text with a phosphor cursor, per-seat telemetry (token count from usage; `~TTFT`/`~tok/s` marked approximate — client-derived), and a **latch** animation on stance-filed (hard snap + glitch, frame flips green; red on abstain). Mediator gets a distinguished cyan system panel. Token streaming stays best-effort display (existing `StreamSink` drop-under-load is unchanged; UI does NOT claim to reload authoritative text). **Stream-gap signal:** the `Token` event's `seq` is a *shared session-global* atomic (`lib.rs:142`) — it interleaves seats, so it can't detect per-seat gaps. Add a separate per-sink `seat_seq` (each `StreamSink` counts from 0, reset per attempt; global `seq` left untouched for the stale-fence). Gap detection = `seat_seq` contiguity, baseline reset on `seat_started`. Contiguity alone misses a dropped *final* token or a wholly-dropped stream, so `SeatUsage` also carries `emitted_seat_chunk_count` (how many chunks the sink emitted for the accepted attempt; `0` is unambiguous — covers the `completion.text`-without-streamed-chunks fallback at `lib.rs:548`). A seat is marked **incomplete** unless the client received exactly that many contiguous chunks. On any gap, render "⚠ stream incomplete". Tests: interleaved seats → no false gaps; final-token drop, whole-stream drop, and a zero-chunk successful completion each behave correctly.
6. **Event-log rail (new component).** A bounded ring buffer (cap ~N lines, oldest evicted) rendering **lifecycle events only** — `state_changed`, `round_started/complete`, `seat_started`, `stance`, `seat_abstained`, `seat_truncated`, `ruling`, `consensus_downgraded`, `awaiting_user`, usage — as log lines timestamped with **local receipt time** (ordered by receipt; no claim of engine-authoritative ordering). **Token events are excluded** (too numerous, duplicate the transcript).
7. **Convergence readout (new component).** Persistent status-bar meter/oscilloscope driven by **backend truth, not stance prose**: it consumes a single **post-guard** per-round telemetry event (see step 9) carrying `effective_ruling` + `cluster_fraction` + `mean_confidence`, and folds `consensus_downgraded` live within a round. It never trusts the pre-guard `Ruling` event. Tightens toward alignment as the cluster grows, snaps to locked on effective `CONSENSUS`, jitters red on `DEADLOCK`. Replaces the old `warmth` glow. Never infers consensus from stance strings.
8. **Verdict finale (rebuild `VerdictScreen`).** On ruling, run a fake compile/boot sequence (ASCII progress + streaming log lines) resolving into the verdict slammed on-screen in Departure Mono; finalization phase labeled distinctly ("SYNTHESIZING…"). Export re-styled as a terminal "session dump"; same markdown payload via `exportMarkdown`.
9. **Real token telemetry (surgical Rust).** `Completion` carries `usage`, dropped by the engine at `crates/krunch-engine/src/lib.rs:547`. Work:
   - **Fix Anthropic input parsing:** `sse.rs` sets `input_tokens: None` — Anthropic reports input on `message_start.usage.input_tokens` (not `message_delta`). Parse + retain it (preserve unknown-vs-zero — usage fields stay **nullable** on the wire), add a provider fixture. OpenAI already yields both.
   - Add `EngineEvent::SeatUsage { session, round, seat, attempt, input_tokens: Option<u32>, output_tokens: Option<u32>, emitted_seat_chunk_count: u64 }`, emitted **only after `accept_attempt(attempt_id)` succeeds** (`lib.rs:556`) — never for a discarded/retried attempt. `emitted_seat_chunk_count` (0 unambiguous) lets the client detect final-token / whole-stream drops (see step 5). `attempt` (consistent with `SeatStarted`/`Token`) lets the store dedupe by `(round, seat, attempt)`; retries never double-count.
   - Emit a **single post-guard** per-round telemetry event carrying `effective_ruling` + `cluster_fraction` + `mean_confidence` (computed for non-Consensus rulings too). Emit it **before** `RoundComplete` (between `lib.rs:394` and `:397`) so the store has the effective ruling/metrics before it snapshots the round; the store stashes pending metrics and `snapshotRound` consumes them keyed by round id. (Event-ordering test.)
   - Add a per-sink `seat_seq` to the `Token` event (see step 5) — reset per attempt, for gap detection only; leave the shared global `seq` alone.
   - Mirror in `types.ts`; handle in the store (per-seat + session totals). Totals labeled `exact` only when every included completion reported both counts, else `partial`; never sum nulls as zero. The figure is **"accepted-completion tokens/cost"**, never "total billing" — discarded/retried attempts may have consumed provider tokens not counted here. **Transient UI-only for v1** — resets on reload; no SQLite migration (persisted read-model deferred).
   - `MODEL_PRICES`: parse the seat `base_url`, normalize the origin, and price only when it exactly equals `https://api.openai.com` (OpenAI) or `https://api.anthropic.com` (Anthropic). Any other origin, `claude_cli`/`codex_cli`/`demo`, or unknown model → `—`, never fabricated.
10. **Boot sequence.** On app launch, a brief ASCII boot/self-test animation resolves into the cockpit (skippable, reduced-motion → instant).
11. **Accessibility & window.** Ship a persistent **effects/contrast toggle** (Off / Ambient / Max, default Ambient) in `localStorage`; static scanline/glow opacity is capped so body text clears a contrast floor even at Ambient. `prefers-reduced-motion` still kills all motion (flicker/glitch/type-on/latch/shake). **Compact desktop layout:** at/near the 940×640 min window the log rail collapses and the seat grid scrolls; verify the full six-seat + mediator + rail + palette + overlay layout at 940×640 and at 200% zoom.
12. **Verify.** Browser preview (`?preview=` seeds): each phase, no console errors, streaming legibility, latch/convergence/verdict animations, palette + shortcut guards, reduced-motion, and min-window/200%-zoom. **Plus tests the preview can't reach:** Rust unit tests for Anthropic `message_start` input-usage parsing and `SeatUsage` emit; frontend store tests for usage dedupe across retries, log-rail cap/eviction, shortcut focus/IME guards, and reduced-motion branches.

## Key decisions & tradeoffs
- **Total replacement, not evolution.** The entire "Deliberation Chamber" identity (brass, Young Serif, calm) is discarded. High blast radius: every screen + `style.css` + fonts change at once.
- **One persistent cockpit** replaces three screens; `store.phase` now drives in-place pane morphing rather than screen swaps. Keeps store logic intact but concentrates layout complexity in the shell.
- **Multi-color tactical palette** — color encodes status. Must still clear a contrast floor on near-black; flicker/glow must not push text below legibility during long reads.
- **Mono everywhere**, including long agent-reasoning prose set in Fragment Mono — chosen for terminal authenticity over the marginally better readability of a grotesk for long text.
- **All three signature animations layered** (convergence readout + seat latch + verdict boot). Richest but noisiest; the "ambient always-on, big effects only on discrete events" rule is the guardrail against constant motion.
- **Real tokens via one new engine event.** Surgical but touches tested engine code and the serde wire contract; must not break existing `krunch-engine`/store handling. Anthropic input usage requires new `message_start` parsing (currently dropped). `SeatUsage` carries `attempt` and is deduped by `(round,seat,attempt)` so retries don't double-count. Usage is **transient UI-only for v1** (resets on reload; no persistence/migration this pass).
- **Approximate rates, exact counts.** Token *counts* are exact (from provider usage); *rates* (`~tok/s`, `~TTFT`) are client-derived and explicitly marked approximate — no engine timing pipeline in v1.
- **Client-side price map for $** — keyed by canonical `(provider, model)`, hardcoded rates go stale; est. $ shown only for known OpenAI/Anthropic models, `—` for custom endpoints / CLI / demo / unknown. Never fabricated.
- **Convergence is backend-driven.** The readout uses ruling state + mean confidence + a backend cluster metric, never inferred from free-text stance prose (which the frontend can't cluster).
- **Effects gated behind an accessibility toggle**, not just reduced-motion — static scanline/glow still affects contrast, so Off/Ambient/Max is a persistent user preference with an accessible default.
- **Cuts — decided (user: "dealer's choice, make it pretty and useful").** No feature removed. Adopt all three restructures: (a) verdict becomes a dismissible overlay over the cockpit, not a separate route; (b) the roman-numeral setup ceremony collapses into one dense config pane; (c) `AwaitingUserModal` becomes an inline console prompt in the log rail (non-blocking, power-user). Full capability preserved, just re-expressed.

## Performance budget
Type-on rendering is batched via `requestAnimationFrame`; per-seat text repaint capped (~≤30fps). **Real worst case = 6 concurrent panelist streams** (panelists run under `join_all`, then the mediator streams *after* — never simultaneous; a 7-stream figure is synthetic-only). Targets: ~60fps idle, ≥30fps at 6 concurrent panelist streams, verified at the min window.
Auto-degrade policy: sample frame time over a 1s rolling window; degrade to reduced effects (scanline/glow/glitch off, type-on → instant) after 2 consecutive windows with mean frame > 24 ms (~<42fps); recover after 3s sustained < 18 ms (hysteresis prevents flapping). Precedence: manual **Off** always wins; manual **Max** opts out of auto-degrade; **Ambient** (default) is the only mode auto-degradation touches.

## Risks / open questions
- **Transient telemetry (decided for v1, flagged):** token/cost totals reset on reload — no v1 persistence. Persisted session read-model is the deferred follow-up.
- **Global-hotkey collisions** with native Tauri/webview shortcuts (e.g. `⌘K`, `⌘W`) — verify on macOS during implementation.
- **Verdict boot finale timing:** must be tight + skippable so it never wastes a power user's time.

## Out of scope
- Changes to deliberation logic, consensus/quorum math, provider protocols, or credential handling. The only engine changes are additive: `SeatUsage` event, a post-guard per-round telemetry event, a `seat_seq` field on `Token`, and the Anthropic `message_start` input-usage parse fix. **No persistence-schema change** (usage is transient UI-only in v1).
- Light theme / theming system (terminal is dark-only, like the old app).
- New providers, models, or backend features unrelated to token-usage telemetry.
- Mobile/responsive layouts — this is a desktop Tauri power tool.
- Session replay/scrub and seat presets (noted as possible future power-user extensions, not v1).
