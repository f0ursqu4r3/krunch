# krunch — design system

## Design Context

### Users
Technically-literate people (developers, researchers, decision-makers) who convene a panel of LLMs to deliberate a hard question and want a considered verdict — not a single model's take. They run multi-round sessions at a desk, often watching a live deliberation unfold over minutes. They value seeing *how* the panel reasoned, not just the answer.

### Brand Personality
Live · high-stakes · instrumented. krunch is a **deliberation ops terminal** — a live signal room, not a chat app. It should feel like watching a mission console during a run that matters: channels lit up, a trace on the scope, everything instrumented and legible under pressure. Three words: *live, monitored, decisive*.

### Aesthetic Direction — "Signal"
- **Theme: pure black void** (`--bg-deep`), surfaces lifted just enough to read, neutrals carrying a faint green cast (hue ~165) — never warm.
- **Accent: neon green signal** — the single primary color, phosphor-bright. Deliberation states ride the same hue family: bright green as the panel converges (`--consensus`), dim signal while it runs (`--continue`), neon red at deadlock/alert (`--deadlock`, `--danger`).
- **Signature moment: Oscilloscope Sync.** Each panelist is a waveform channel — amplitude tracks confidence, streaming seats jitter. As `clusterFraction`/`meanConfidence` climb, channels phase-lock into one bright-green trace ("SIGNAL LOCK" on CONSENSUS); on DEADLOCK they desync into red noise. Driven entirely by live convergence data from the store, not decorative.
- **Panelists are channels:** seats read as `[ CH01 ]`-style channel strips in a HUD, each a voice you can watch stream in real time.
- **The verdict is a decoded transmission** — surfaced as sealed, monospaced record output, exportable as a filing.
- Anti-references: warm/old-world, parchment, serif gravitas, candlelight, skeuomorphism, generic AI dark-mode (cyan-on-black, purple gradients, glassmorphism), shadcn-default indigo/zinc, sterile SaaS dashboards.

### Palette (tokens)
- `--bg-deep` — pure black; the void behind everything.
- `--bg`, `--surface` / `--surface-2` / `--surface-3` — green-cast graphite (hue ~165), lifted in steps for panels/chrome.
- `--fg` / `--fg-muted` / `--fg-faint` — phosphor ink, dimming for secondary/tertiary text.
- `--signal` / `--signal-bright` / `--signal-deep` — neon green (hue 160), the single accent. Renamed from the old `brass` token; same role, new hue.
- `--consensus` — bright green, the panel locking in.
- `--continue` — dim signal, the panel still running.
- `--deadlock` / `--danger` — neon red (hue 22), alert and failure state.

### Type — Monaspace family only
- **Departure Mono** (pixel) — big headings, wordmark, verdict title. Used large and sparingly.
- **Monaspace Neon** (`--font-sans`) — UI / chrome: labels, buttons, body of the interface.
- **Monaspace Krypton** (`--font-mono`) — telemetry: channel ids, confidence %, round counters, model names, markdown headings.
- **Monaspace Argon** (`--font-prose`) — streamed deliberation prose; the humanist cut, legible for long-form reading.
- **Monaspace Xenon** (`--font-record`) — verdict / record body; the slab cut for the sealed transmission; also markdown code (inline + fenced).
- (Deliberately NOT Young Serif/Hanken Grotesk from the old chamber identity, and not Inter/DM/Fraunces/Playfair/Cormorant/Space/IBM Plex.)

### Effects kit
- **Phosphor glow/bloom** (`.glow-text`, `.glow-edge`) — neon elements bleed light; intensity scales with `--effects-intensity`.
- **Moving grid + dataflow** (`.hud-grid`) — a drifting dot/line grid plus a diagonal dataflow sweep behind the UI; GPU-cheap transform/position animation.
- **HUD chrome** (`.terminal-panel`) — notched panels via `clip-path`, hairline borders, a corner bracket drawn with `::before`, bracket-style labels (`[ CH01 ]`).
- **Glitch** (`.glitch`) — a brief RGB-split shudder on state changes (e.g. DEADLOCK) and errors.
- All of the above are gated on `--effects-intensity` (continuous 0–1, exposed as **Off / Ambient / Max** in the top bar), and further collapse under `prefers-reduced-motion` and a performance auto-reduce. At Off/reduced motion, the oscilloscope freezes on a single static frame (`store.instantTokens`) instead of animating.
- **Deliberately NOT used:** scanlines, CRT curvature/vignette. The terminal reads clean and modern, not a retro monitor pastiche.

### Design Principles
1. **The signal is live.** State is telemetry — glow, grid motion, and the oscilloscope trace track the deliberation in real time, not just badges.
2. **Instrumented over decorated.** Weight comes from real data driving real visuals (confidence → amplitude, convergence → phase-lock), not gratuitous chrome.
3. **Everyone has a channel.** Panelists are waveform channels and HUD strips, each a voice you can watch stream and converge — or not.
4. **A verdict is a decoded transmission.** Outcomes read as sealed, monospaced record output and filed transcripts, not toast notifications.
5. **Calm under load.** Characterful but legible across long multi-round sessions; motion serves comprehension and degrades gracefully under reduced motion or low performance.
